# 声笔 VoiceDraw — 设计文档（前后端系统版 · v2）

> 纯语音控制的创作工具。不用鼠标、不用键盘，说出指令即可绘制、编辑、排版。
> 仓库：https://github.com/heziamo/voicedraw ｜ 线上：https://voicedraw.duckdns.org:8443/（Let's Encrypt 绿锁）

升级历程：v2 把前端从「单页画图 demo」重做为**专业创作工具界面**（图层面板 + 属性检查器 + 画布工具栏），并把指令能力扩到 **20 种图形 + 6 套复合模板 + 层级/透明度/翻转/对齐/分布等专业编辑**；**v3 接入文生图大模型**（智谱 CogView），说「生成一张星空下的城堡」即可让大模型出图，生成的图片作为一等对象进画布，可被所有编辑指令操作；未配置 Key 时回退占位图，链路照常跑通。**v4 加入 LLM 兜底理解**（规则没听懂时交 GLM 纠正识别错字/口语再执行）+ 前端浅色主题；**v5 把「画实物→AI 出图」做成确定性路由**——「画一条鱼」这类既非图形也非模板的实物，直接走文生图，不再依赖弱模型判断，确保每条指令都能落地。

## 1. 项目目标与约束

| 维度 | 要求 | 应对策略 |
|---|---|---|
| 交互方式 | 仅语音，禁用鼠标/键盘 | 全部操作走「识别 → 后端解析 → 执行 → 语音播报」闭环；UI 上的工具栏按钮仅是**冗余可达性入口**，点击也只是触发与语音完全相同的指令路径 |
| 指令理解准确性 | 容错口音、谐音、口语化 | 谐音矫正表 + 同义词词典 + 礼貌语剥离 + 中文数字解析 + 默认值兜底 |
| 响应延迟 | 越低越好 | 后端纯本地规则解析，实测端到端 **4–6ms**（局域）；瓶颈只剩浏览器 ASR |
| 复杂指令 | 拆解与顺序执行 | 后端分句器把一句话切成多条原子指令，逐句解析；多对象操作按质心整体变换保持构图 |
| 工程化 | Python 前后端 + Git + 部署 | FastAPI + 原生前端，GitHub 版本管理，systemd + TLS 部署到云服务器 |

**唯一的非语音操作**：首次进入需点一次「开始语音创作」。浏览器安全策略硬性要求麦克风由用户手势开启，无法绕过；此后所有操作均为语音。工具栏的撤销/缩放/网格按钮是给视觉反馈用的冗余入口，删掉它们应用依然 100% 可用。

## 2. 系统架构

```
┌─────────────────── 浏览器（瘦客户端） ───────────────────┐
│ 顶栏：撤销/重做态 · 会话/麦克风状态                          │
│ ┌─图层面板─┐ ┌──── 画布 ────┐ ┌─语音 + 属性检查器 + 日志─┐ │
│ │对象列表  │ │工具栏:缩放/网格│ │ 实时字幕/选中对象属性    │ │
│ │字形+色块 │ │Canvas 渲染   │ │ 指令日志(带后端延迟)     │ │
│ └─────────┘ └──────────────┘ └────────────────────────┘ │
│  麦克风 → Web Speech 识别 → final 文本                       │
└────────────────────────────┬──────────────────────────────┘
                final 文本     │ HTTPS POST /commands
                场景 + 结果     │ JSON
┌────────────────────────────▼──────────────────────────────┐
│ FastAPI 后端（Python 3.8+）                                  │
│  NLU 流水线（纯标准库，<5ms）                                 │
│   归一化 → 分句 → 意图解析(32) → 指代消解 → 任务规划 → 执行器  │
│  会话状态机（场景图 + 撤销/重做栈 + 选中集 + 组）              │
│  可插拔持久化（SQLite / JSON 文件）+ 指令审计日志             │
└────────────────────────────┬──────────────────────────────┘
                   SQLite(本地) / JSON 文件(服务器)
```

### 2.1 前后端职责划分

| 层 | 归属 | 职责 |
|---|---|---|
| 语音采集与识别 | **前端** | 只有浏览器能访问麦克风；Web Speech API 做 ASR |
| 渲染 + 界面状态 | **前端** | Canvas 2D 画 20 种图形；图层面板/属性检查器从场景图派生；缩放、网格、PNG 导出 |
| TTS 播报 | **前端** | SpeechSynthesis；回声抑制（播报时挂起识别） |
| **NLU 解析** | **后端** | 词典 + 规则把中文转结构化操作（核心，33 个意图） |
| **状态管理** | **后端** | 场景图、撤销/重做、选中集、组；前端无状态，刷新可从服务器恢复 |
| **持久化与审计** | **后端** | 会话落库，每条指令记审计日志 |

把 NLU 与状态放后端：① 解析逻辑可被任何客户端复用；② 词典迭代不必让用户刷新；③ 刷新恢复、跨设备续画、审计回放天然成立。代价是每条指令一次网络往返，但解析 <5ms，相比 ASR 的 0.3–1.5s 可忽略。

### 2.2 前端界面设计（v2 重点）

初版被指出「AI 味重、功能单一」——根因是用了**营销落地页**的版式（居中大标题、三段式卡片、信息稀疏）。v2 改为**真实创作工具的应用 chrome**，参考 Figma / Linear 桌面端：

- **三栏密排布局**：左 248px 图层面板，中弹性画布，右 312px 检查器/语音/日志。
- **图层面板**：场景图的对象列表，顶层在上；每行 = 字形图标 + 名称 + 透明度 + 色块；复合模板折叠成一行并标「N 部件」；选中行有薰衣草左边框。
- **属性检查器**：选中单个对象时以等宽数字显示 类型/位置 X·Y/尺寸/颜色（带色块+hex）/旋转/透明度/填充/翻转；多选时显示对象构成与可用批量操作。
- **画布工具栏**：缩放百分比 + 适应 + 网格开关 + 画板尺寸 + 「仅语音操作」徽标。
- **设计语言**：沿用 Linear 令牌（近黑 `#010102`、四级 surface、发丝线、唯一强调薰衣草蓝 `#5e6ad2`），但应用为密排工具 chrome 而非落地页。白色画板带阴影/边框，悬浮在深色工作区上（同 Figma 的画板隐喻）。等宽 JetBrains Mono 承载所有数值读数。

界面上能看到的所有控件都有对应语音指令；工具栏按钮被显式定位为「冗余可达性」，不破坏纯语音约束。

### 2.3 关键设计决策

1. **规则优先、LLM 兜底的混合 NLU** —— 干净指令走本地规则（<5ms、零成本、可预测）；规则没听懂才调大模型纠错（见 §2.6）。两者结合：日常快、长尾稳。
2. **保留模式场景图** —— 每个图形是带 id 的对象，支撑事后修改类指令（改色/移动/层级…）。
3. **逻辑坐标系 1200×750** —— 服务端不感知客户端像素，前端 contain 等比缩放，跨设备一致。
4. **组与整体变换** —— 复合模板的部件共享 `group`，整组选中；多对象的移动/缩放按**质心**整体平移/等比缩放，保持构图不散架。
5. **可插拔持久化** —— 服务器源码版 Python 3.8 缺 `_sqlite3`，`store.py` 自动回退 JSON 文件后端，无需重编译 Python。
6. **TTS 回声抑制 + 静音不关麦** —— 播报时挂起识别；「暂停聆听」只在后端静音（仍听「继续聆听」），否则用户无法用语音恢复。

### 2.4 HTTP API

| 方法 | 路径 | 说明 |
|---|---|---|
| `GET` | `/api/healthz` | 健康检查（版本 + 存储后端） |
| `POST` | `/api/sessions` | 新建会话 |
| `GET` | `/api/sessions/{id}` | 取场景（刷新恢复） |
| `POST` | `/api/sessions/{id}/commands` | **核心**：提交语音文本，返回新场景 + 每条子句结果 |
| `GET` | `/api/sessions/{id}/commands` | 指令审计日志 |

响应：`{session_id, scene{shapes,bg}, selected[], flags{muted,tts_on,can_undo,can_redo}, logical{w,h}, results[{ok,msg,intent,action,clause}]}`。

### 2.5 文生图（AI 大模型）

```
「生成一张星空下的城堡」
  → 解析阶段：generate 意图抽取提示词，返回 action=generate_image（不阻塞，<5ms）
  → 前端弹「生成中」遮罩，调 POST /generate（独立慢端点）
  → 后端 imagegen.generate(prompt)：
       有 Key → 调智谱 CogView，下载图片存 /opt/voicedraw/data/images/
       无 Key → 生成 SVG 占位图（同样存盘）
  → add_image() 把图片作为 type=image 的对象加入场景图、整张选中
  → 前端按 src 加载位图渲染；此后「把图片放大/旋转/移到右边/删除」等指令照常作用
```

几个关键决策：
- **快慢分离**：指令解析仍 <5ms 返回；耗时的出图走独立 `/generate` 端点，前端配加载遮罩，不卡其他指令。
- **图片是一等对象**：`type=image` 的图形带 `src/aspect/prompt`，复用既有的指代消解与移动/缩放/旋转/翻转/透明度/删除指令——无需为图片单独写编辑逻辑。
- **provider 抽象 + 占位回退**：`VOICEDRAW_IMAGE_PROVIDER` 选服务商（现实现 zhipu）；无 Key 自动出占位图，保证「先把流程做完、Key 随时插」。
- **只用 stdlib urllib**：规避服务器老 OpenSSL 上 requests/urllib3 v2 的兼容坑。
- **图片存盘 + /media 托管**：不塞进场景 JSON（避免 base64 撑爆存储），场景只存 `src` 路径，图片同源托管，导出 PNG 不会污染画布。
- **Key 不进代码/git**：systemd `EnvironmentFile=-/opt/voicedraw/.env`，用户写入 `VOICEDRAW_IMAGE_API_KEY` 重启即启用真实生成。

### 2.6 LLM 兜底理解（解决识别错就失效的脆性）

纯规则解析的硬伤：语音识别错一个字（圆→缘、蓝→篮）或换种口语说法，规则就匹配不上、直接「没听懂」。v4 引入**混合式理解**：

```
一句话 → 归一化 → 分句
  每个子句先用规则解析（<5ms）
    命中 → 直接执行（快路径，绝大多数干净指令走这里）
    没听懂 → 交 GLM：把「可能识别错/口语化」的原话还原成【标准指令字符串】
              → 再用同一套规则解析执行（复用全部已测逻辑）
              → 日志标注「AI 理解为：…」，让用户看到纠错过程
```

例：「画一个红色的**缘**」规则失败 → GLM 还原为「画一个红色的圆」→ 正常画出。「明天天气怎么样」→ GLM 判定非绘图指令、返回 `NONE` → 保持「没听懂」。

几个关键决策：
- **让 LLM 纠错而非直接执行**：LLM 只把口语翻译成本工具的标准指令文本，再交给既有规则引擎——既得到大模型的鲁棒理解，又复用全部已测过的执行/撤销/指代逻辑，不必让 LLM 直接操控场景（更可控、更省 token）。
- **规则优先、LLM 兜底的混合式**：干净指令仍 <5ms，不为鲁棒性牺牲日常延迟；只有长尾/错字才付 LLM 的 0.5–2s。
- **同一把智谱 Key**：`glm-4-flash` 免费，与 CogView 文生图共用 `.env` 里的 Key；缺 Key 时自动退回纯规则（不报错）。
- **few-shot + 场景上下文**：系统提示给出标准指令词汇表和「错字→正字」示例，并附当前画布构成（如「画布现有：圆形×2」），帮助 LLM 理解「把它放大」之类的指代。

### 2.7 「画实物」→ 文生图的确定性兜底（v5）

LLM 兜底的实战教训：免费的 `glm-4-flash` 不够稳——「画一个红色的鱼」它有时原样返回「画一个红色的鱼」而不肯改写成文生图，导致再解析仍「没听懂」。**不能把"画不出的实物该不该转 AI 出图"这种判断押在弱模型身上。** v5 把这条规则下沉到规则引擎，做成**确定性路由**：

```
「画 / 做 / 来一个 + X」
  X 命中已知图形（圆/方/三角…）        → draw   意图，画矢量图形（精确、即时）
  X 命中复合模板（笑脸/房子/树…）       → template 意图
  X 既不是图形也不是模板（鱼/猫/汽车/龙/皮卡丘…）
                                     → draw_object 意图：抽出实物描述当提示词，
                                       直接 action=generate_image 走文生图（0 延迟、不依赖 LLM）
```

几个关键决策：
- **确定性优于智能**：凡「画+任意实物」都必出图，不再有死胡同——直接回应用户「让每条指令都能执行」。比走 LLM 还快（省掉 0.5–2s 往返），且不受弱模型发挥影响。
- **谐音错字归一化层解决**：把「圆」的常见错字（园/原/元/**缘**）放进 normalizer，misheard 的*图形*仍矫正成矢量圆，不会误当实物去 AI 出图（「边缘/缘分」等非词尾不误伤）。
- **三层防线**：① 规则矢量（已知图形/模板，<5ms）→ ② 实物兜底文生图（draw_object，确定性）→ ③ LLM 纠错（真正的口语/乱序长尾）。LLM 退居第三层，只处理规则与确定性兜底都接不住的输入，弱模型抽风也不再阻断主流程。

谐音矫正（花→画、园形/原形→圆形、X型→X形、兰色→蓝色、菱型→菱形、撤消→撤销）·礼貌语剥离·中文数字（到几百，量词扩到 朵/棵/座）·全角→半角·百分之N→N%·同义动词·失败必反馈（播报原文 + 提示「帮助」，日志红标）。

## 4. 延迟分析

| 阶段 | 耗时 | 可控性 |
|---|---|---|
| ASR（说完→final） | 0.3–1.5s | ❌ 浏览器/云端 |
| 网络往返 | 局域 <5ms / 公网数十ms | ⚠️ 取决于部署位置 |
| 后端解析+执行 | **<5ms**（实测整链 4–6ms） | ✅ 纯本地规则 |
| 前端渲染（含图层/检查器重绘） | <16ms | ✅ Canvas 2D + DOM |
| TTS（可关） | 1–3s，期间挂起识别 | ✅ 「关闭播报」 |

应用自身引入延迟 < 一帧；整体体感延迟≈ASR 本身。这是选本地规则而非云端 LLM 的核心理由。

## 6. 未完成项与原因

**6.1 LLM 兜底解析 —— 已实现（v4）**。规则没听懂的子句自动交智谱 GLM（默认免费 `glm-4-flash`）把可能识别错/口语化的话还原成标准指令再执行，详见 §2.6。解决了「识别错一个字就废」的脆性。混合式设计：规则先走快路径（<5ms 命中即返回），仅失败时才调 LLM（多 0.5–2s）——日常指令仍快、长尾表达靠 AI 兜底。`app/llm.py` 与文生图复用同一把智谱 Key。

**6.2 自由手绘曲线**。语音是离散低带宽通道，无法承载连续轨迹控制——交互模态本质限制，非工程量。参数化图形已覆盖绝大多数需求。

**6.3 颜色深浅微调 / 渐变填充**。深浅微调需色彩空间运算（HSL 明度增减），渐变需扩展 paint 模型与渲染。为控单文件复杂度推迟；场景图模型已能承载（加字段即可），属纯增量。

**6.4 真正的组合/解组**。复合模板已用 `group` 实现「整组选中 + 整体变换」，但没有独立的「组合两个图形成新组 / 解组」语音指令，也无嵌套组。需要组生命周期管理与层级树，为控复杂度只做了模板级分组。

**6.5 空间指代（"左边那个圆"）**。需按坐标排序消歧，多候选时口语本身有歧义。「第N个」「选中所有X」两条等价路径已覆盖，性价比不足。

**6.6 同子句无连接词多意图**。分句依赖显式连接词；ASR 通常会为停顿补逗号，自然语流大多能正确切分。完整解决需语义级切分（LLM 路线）。

**6.7 已知限制**
- 首次需点一次开启麦克风（浏览器安全策略）。
- 仅 Chrome/Edge 桌面版（Web Speech API 兼容性）。
- 线上已用 DuckDNS 域名 + Let's Encrypt（DNS-01 校验）签发受信任证书、acme.sh 自动续期，访问 `https://voicedraw.duckdns.org:8443/` 绿锁无警告（用裸 IP 访问才会因名称不符报警告）。
- 组的**旋转**仍是每个部件原地自转（未绕组心公转）；移动与缩放已按组心整体变换。
- 窗口缩放后已有图形坐标不重排（保持逻辑绝对坐标）。

## 7. 部署架构

```
用户浏览器 ──HTTPS:8443──► voicedraw.duckdns.org → 117.50.181.92 (CentOS 7.9 / 2GB)
                            └ systemd: voicedraw.service
                              └ uvicorn(内置 TLS,Let's Encrypt 证书)
                                └ FastAPI(前端静态 + API)
                                  └ JSON 文件存储 /opt/voicedraw/data/
   证书：acme.sh + DuckDNS DNS-01 签发/自动续期（不需开 80/443，绕开未备案的 GFW 重置）
```

- **为什么 HTTPS**：Web Speech API 与 `getUserMedia` 要求安全上下文，HTTP 下无法授权麦克风。uvicorn 内置 TLS 满足，省去 nginx。
- **为什么证书走 DNS-01**：服务器在国内未 ICP 备案，GFW 重置国际到 80/443 的请求，HTTP-01/TLS-ALPN-01 校验必失败；DNS-01 只加一条 DNS TXT 记录、不走入站连接，故可签发受信任的 Let's Encrypt 证书。域名用免费的 DuckDNS（acme.sh 自动管理 TXT 与续期）。
- **为什么 JSON 存储**：服务器源码版 Python 3.8 缺 `_sqlite3`，自动回退。
- **一键部署**：`bash deploy/deploy.sh`（rsync → venv 清华源 → 生成证书 → systemd → 健康检查），开机自启已 enable。
